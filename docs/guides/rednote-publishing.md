# 小红书图文发布(rednote 图卡)

> **适用插件版本**：v3.9.19+ ｜ **最后更新**：2026-07-13 ｜ 编码：UTF-8
> 渲染引擎自 [note-to-red](https://github.com/) 全量移植(原插件保留可独立使用),设置互不影响。

## 1. 功能概览

在「Content Studio」面板内直接**预览、导出、一键发布**小红书图文笔记。
自 v3.9.19 起小红书与公众号交互统一:同一条顶栏,按模式切换按钮组。

```
顶部操作区:[公众号 ▾] 平台下拉        ← 只切换预览(后续新平台在此扩展)
小红书模式顶栏:⚙ 样式设置(模板/主题/字体/字号/背景图/使用指南,悬浮层)
              ⬇ 下载(小菜单:下载当前页 / 导出全部页)
预览区 = note-to-red 图卡渲染 + 底部翻页导航(← 1/N →)
发布:右上角 ✈️ →「发布与分发」窗口 →「其他平台」tab 勾选小红书 → 发送
      (所有平台统一从此窗口发布;勾选小红书时自动走图卡链路)
```

## 2. 笔记格式约定(重要)

一篇可发布的小红书笔记长这样:

```markdown
# 笔记标题(H1 = 小红书标题)

> 发布时复制下面这段作为笔记正文：

这里是发布到小红书的文字正文,可多段。
#话题1 #话题2

> 下面每个二级标题（##）＝ 一张图：

## 第一张图卡🎯
图卡内容…(## 分节,--- 可再分页)

## 第二张图卡🥊
…
```

发布时插件自动:
- **标题** = 第一个 `# H1`(没有则用文件名);
- **正文** = 第一个 `> 发布时复制下面这段作为笔记正文：` 之后、下一个 `>` 行之前的内容(不含标记行);找不到标记会**直接报错**,不发。
- **图片** = 当前小红书预览渲染出的**全部图卡**(所见即所发)。

## 3. 发布流程

1. 打开笔记 → 发布助手面板 → 顶部平台下拉选 **「小红书」**,点顶栏 **⚙ 样式设置** 调好模板/主题/字体/字号/背景,翻页确认每张图卡;
2. 点右上角 **✈️** 打开「发布与分发」→ **「其他平台」tab** → 勾选 **小红书**(可同时勾其他平台)→ 点「发送到浏览器插件」;
3. **勾选了小红书就自动走图卡链路**(其他平台照常走文字链路),插件依次:
   - 渲染全部图卡(PNG,4x 像素;若没打开过小红书预览会自动切换渲染);
   - 在笔记同目录创建 **`sync-to-rednote/`**(已存在则**先清空**),写入 `synced-rednote-card_00.png、synced-rednote-card_01.png…` 留档(发布后**不删除**);
   - 截取标题/正文,图卡作为附件,经**浏览器插件桥接**投递到小红书;
   - 图卡链路失败(如缺正文标记)只**跳过小红书并提示原因**,不阻断同批其他平台;
4. 到浏览器插件任务窗口 / 小红书创作者中心**草稿箱**检查排版,手动发布。

## 4. 前置条件

- 设置 →「其他平台」已**开启浏览器插件发布**并配对成功(连接令牌一致);
- 浏览器扩展在线,且平台列表里能看到**小红书**(没有就点「测试连接」刷新);
- 那个 Chrome 里已**登录小红书**;
- 免费版扩展每天有平台额度限制,发布失败也计额度,注意节约。

## 5. 设置项

设置 → 微信 tab → 折叠区 **「小红书图卡(Note to RED)」**:
用户信息(头像/昵称/ID/时间/页脚)、标题分割级别(H1/H2)、**主题管理**(新建/编辑/删除/显隐)、**字体管理**。所有设置存本插件 `data.json` 的 `rednote` 命名空间,与 note-to-red 原插件互不影响。

## 6. 常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| 报「未找到正文标记」 | 笔记里没有 `> 发布时复制下面这段作为笔记正文：` | 按第 2 节格式补上标记与正文 |
| 报「平台列表里没找到小红书」 | 扩展平台列表未同步 | 设置 →「其他平台」→ 测试连接 |
| 报「请先开启浏览器插件发布」 | 多平台同步开关未开 | 设置 →「其他平台」开启并配对 |
| 图卡是空白/缺节 | 笔记没有二级标题分节 | 用 `##` 分节(每节一张图) |
| 投递成功但草稿没出现 | 扩展后台被杀/网络问题 | 参考多平台同步排查(保持 Chrome 前台;国内直连小红书) |
