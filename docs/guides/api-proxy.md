# API 代理设置

> **适用插件版本**：v2.9.4 ｜ **最后更新**：2026-07-07 ｜ 编码：UTF-8

## 1. 解决什么问题

微信公众号「素材 / 草稿」API 要求把**调用方 IP** 加进后台白名单。如果你在家里、公司、移动热点之间切换，本机公网 IP 一直变，白名单跟不上，插件同步就会报错（如 `40164 / IP 不在白名单`）。

配置一个**固定公网 IP 的中转代理**后，请求路径变成：

```
Obsidian 插件  ──►  代理服务（固定 IP 的服务器）  ──►  api.weixin.qq.com
```

微信后台只需把**代理服务器的 IP** 加进白名单一次即可，之后本机 IP 怎么变都不影响。

## 2. 在哪里填

Obsidian 设置 → Content Studio →「微信」标签页 → 拉到最下面 **高级设置 → API 代理地址**。

填入形如：

```
https://<你的代理域名>/proxy?token=<你的访问口令>
```

**要求**：

- 必须是 `https://`（插件会拒绝 `http://`，以保护 AppSecret）；
- 路径是 `/proxy`；
- 如果代理服务端设了访问口令，就带上 `?token=<口令>`，否则可省略。

留空 = 不走代理（直连微信，用本机 IP）。

### 测试代理是否生效

填好地址后，点旁边的 **「测试代理」** 按钮：

- ✅ `代理生效：请求已经代理转发到微信并收到响应` —— 代理连通、转发正常，可以用了；
- ❌ `代理测试失败：...` —— 看提示：401 多半是 `?token=` 不对，网络错误多半是地址/证书/网络问题。

> 测试用的是哑凭证的哑请求，只验证"代理这条链路通不通"，不涉及你的真实公众号账号。

### 同步时怎么确认走了代理

同步过程中，Obsidian 右上角的提示会标注当前通道：

- `🚀 正在使用 xxx 同步...（经 API 代理）` / `✅ 同步成功！...（经 API 代理）` —— 走了代理；
- `...（直连微信）` —— 没走代理（本机直连）。

## 3. 本项目已部署的代理（自用）

已在个人 ECS 上部署好一台自用代理：

| 项 | 值 |
|----|----|
| 代理地址 | `https://wechat-proxy.runfast.xyz/proxy?token=<你的PROXY_TOKEN>` |
| 代理服务器 IP（加进微信白名单的就是它） | `115.190.207.149` |
| 访问口令 `PROXY_TOKEN` | 部署时生成，存在 ECS 的 `/var/www/wechat-proxy/.env`，**不入 git** |

> 把上面的 `<你的PROXY_TOKEN>` 换成 ECS `.env` 里 `PROXY_TOKEN` 的实际值，粘进插件即可。

## 4. 完整启用步骤

1. **微信公众平台** → 设置与开发 → 基本配置 → IP 白名单 → 加入代理服务器 IP（本项目为 `115.190.207.149`）。
2. **插件** → 高级设置 → API 代理地址 → 填 `https://wechat-proxy.runfast.xyz/proxy?token=<口令>`。
3. 回插件正常同步草稿，请求即走代理转发。

## 6. 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 插件报「必须使用 HTTPS」 | 代理地址填了 `http://` | 改 `https://`，确认代理已配好证书 |
| 插件报 401 | `?token=` 没带 / 带错 | 核对 ECS `.env` 里的 `PROXY_TOKEN` |
| 插件报 400「非法 URL」 | 地址填错（不是 `.../proxy`） | 末尾必须是 `/proxy` |
| 微信报 `40164` / IP 不在白名单 | 代理服务器 IP 没加白名单 | 微信后台白名单加 `115.190.207.149` |
| `dig` 显示域名解析成 `198.18.x.x` | 本机在跑 fake-ip 代理（Clash/Surge） | **正常现象**，是透明路由，不是故障；能 `curl` 通就行，别加 hosts |
