# AI Provider 设置

> **适用插件版本**：v2.9.4 ｜ **最后更新**：2026-07-07 ｜ 编码：UTF-8

## 1. 设计思路：一份凭证，两个能力

插件有两个 AI 能力，它们**共用同一份凭证**（AI Provider），各自独立开关和选模型质量：

```
AI Provider（共享凭证，当前 DeepSeek）
   ├── AI 编排（排版）    —— 独立开关 + 模型质量
   └── 标题 AI 润色       —— 独立开关 + 模型质量
```

- **Provider** 只负责“连哪个大模型、用什么 Key”（配一次）。
- **模型质量**（Pro / Lite）由每个能力**在各自区块里选**，Provider 层只标明模型家族 `DeepSeek V4`。

## 2. 第一步：配置 AI Provider（必做）

设置 →「微信」标签页 → **AI Provider** → 点「+ 添加 AI Provider」，按下表填：

| 字段 | 填什么 |
|------|--------|
| 名称 | 随意，如 `DeepSeek` |
| 类型 | **OpenAI 兼容接口** |
| Base URL | `https://api.deepseek.com/v1`（新建时已默认填好） |
| API Key | 你的 DeepSeek API Key（[开放平台](https://platform.deepseek.com/)获取） |
| 模型 | **DeepSeek V4**（唯一项，具体 Pro/Lite 在下面各能力里选） |
| 启用 | 打开 |

保存后，在 **默认 AI Provider** 下拉里把它**选为默认**。两个 AI 能力都会复用这个默认 Provider 的 Key / Base URL。

> 当前只支持 DeepSeek。后续接入其他模型再扩展类型/模型列表。

## 3. AI 编排（排版）

作用：让 LLM 根据文章内容，一键生成并应用一套公众号排版（布局 + 配色）。

设置 → 折叠区 **「AI 编排」** 展开：

| 设置项 | 说明 |
|--------|------|
| 启用 AI 编排 | 关闭后隐藏转换器工具栏里的「AI 编排」入口（缓存不删） |
| 模型质量 | **Pro**（质量优先）/ **Lite**（快、省），复用默认 Provider |
| 默认布局 | 打开面板时默认选中的布局，`自动推荐` 让 AI 挑 |
| 默认颜色 | 默认配色，`自动推荐` 让 AI 挑；生成后可手动切色 |
| 编排时参考图片 | AI 把文中配图当排版素材参考（不改正文） |
| AI 请求超时 | 默认 120 秒 |
| AI 编排缓存 | 已生成结果的缓存，可一键清空 |

**使用**：打开转换器面板 → 顶部工具栏「AI 编排」按钮 → 选布局/颜色 → 生成 → 应用到预览。

## 4. 标题 AI 润色

作用：在发布窗口的标题旁一键让 LLM 根据正文优化标题（给 5 个候选），并自动去除标题里的 `(MM-DD-HHMM)` 时间码。

设置 → 折叠区 **「标题 AI 润色」** 展开：

| 设置项 | 说明 |
|--------|------|
| 启用标题 AI 润色 | **关闭后，发布窗口标题旁不再显示「✨ AI 润色标题」按钮** |
| 模型质量 | **Pro** / **Lite**，复用默认 Provider |

**使用**：发布与分发窗口 → 标题旁点「✨ AI 润色标题」→ 从 5 个候选里点一个采用。

## 5. Pro 与 Lite 怎么选

| | DeepSeek V4 Pro | DeepSeek V4 Lite |
|---|---|---|
| 定位 | 质量优先 | 快、省 |
| 适合 | 排版编排、重要标题 | 量大、追求速度/成本 |

两个能力可以各选各的（比如编排用 Pro、标题用 Lite）。

## 6. 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 标题旁没有润色按钮 | 「启用标题 AI 润色」关着 | 到折叠区打开 |
| 提示「尚未配置默认 AI Provider」 | 没加 / 没设默认 Provider | 先按第 2 步加一个并设为默认 |
| 工具栏没有「AI 编排」 | 「启用 AI 编排」关着 | 到折叠区打开 |
| 调用报鉴权失败 | API Key 错 / 没填 | 检查 Provider 的 API Key |
| 出网被公司代理 TLS 重置、调不通 | 客户端直连大模型撞公司代理 | 换住宅网络 / 热点重试 |
