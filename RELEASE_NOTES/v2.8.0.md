---

## 更新内容

### 🚀 重大更新

- **浏览器插件安全握手**：连接建立时强制进行 `extension_hello` 握手，通过配对令牌 + 实例 ID 双重验证，杜绝非预期连接。旧版握手路径已移除。
- **多客户端支持**：支持多个浏览器实例同时连接，自动路由到主连接客户端，并在设置页展示各连接状态。
- **心跳确认（heartbeat_ack）**：Obsidian 现在会回应浏览器插件的心跳帧，使浏览器插件能准确检测插件是否存活并在必要时重连。
- **Quota 策略传递**：发布时自动向浏览器插件传递 `quotaPolicy`，免费用户超出平台配额时自动截断而非阻断整批发布。

### 🐛 问题修复

- **修复设置页死锁**：修复首次安装时「启用浏览器插件发布」开关因令牌为空而被禁用，导致无法开启、也无法填写令牌的死锁问题。现在开关始终可点击，开启后再填写令牌即可。
- **修复本地图片双重编码**：通过 Bridge 发布时本地图片不再被重复 Base64 编码，解决部分平台图片显示异常。
- **修复封面资源处理**：正确识别并转换 `asset://` 格式的封面图片，生成缩略图后传递给浏览器插件。
- **修复同实例 ID 重连风暴**：相同实例 ID 的扩展重连时直接接管会话，不再累积 backoff 重试导致 Service Worker 重载风暴。
- **Token 不匹配提示**：连接令牌校验失败时显示明确操作提示，引导用户复制新令牌粘贴到设置页。

### ✨ 功能优化

- **浏览器图标识别**：设置页连接状态栏展示对应浏览器品牌图标（Edge、Firefox、Brave、Opera、Vivaldi、Arc 等），Chrome/Chromium 使用通用图标避免误认 fork 版本。
- **连接状态栏**：统一展示当前已连接的浏览器实例、上次连接时间和连接状态，替代原来分散的状态指示。
- **连接客户端注册表上限**：注册表最多保留 20 条记录，自动淘汰最旧的断开连接记录，防止长期运行下内存无限增长。

### 🧪 测试

- 新增心跳确认、quotaPolicy 透传、连接客户端注册表上限、封面资源解析等回归测试。
- 新增设置页 UI smoke test，确保关键配置项不会在重构中被意外移除。
